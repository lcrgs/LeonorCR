WorldEdit

Extending the WorldEdit platform for Minecraft.

Java | Minecraft Modding | Agile Development | Reverse Engineering

My Role

As part of the Software Engineering course, I was part of a team tasked with extending the WorldEdit platform, a popular Minecraft mod. Our mission was to reverse engineer the existing codebase, identify design patterns and code smells, and implement new functionalities. I was responsible for reverse engineering, design pattern identification, and implementing one of the user stories.

Challenges & Strategy

The main challenge was understanding the existing codebase of WorldEdit, which is a large and complex Java project. We had to identify design patterns, analyze code metrics, and refactor code smells while ensuring that new functionalities were seamlessly integrated. Additionally, we followed a Scrum-inspired approach to manage the project, which required careful coordination and communication within the team.

  • Solution: We used tools like MetricsTree and IntelliJ IDEA to analyze the codebase and identify patterns and smells.
  • Solution: We implemented new functionalities by following Agile principles, with weekly sprints and regular code reviews.
  • Solution: We documented our findings and progress in detailed reports, including use case diagrams, class diagrams, and sequence diagrams.

Results & Impact

  • Functional Extensions: We successfully implemented three new user stories, enhancing the WorldEdit platform with new functionalities.
  • Improved Codebase: We identified and refactored several code smells, improving the maintainability of the codebase.
  • Learning Experience: This project significantly improved my understanding of software engineering principles, Agile methodologies, and reverse engineering.
