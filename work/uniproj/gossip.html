Gossip!

Simulating the spread of gossip in a community.

Java Programming | Object-Oriented Design | Simulation

My Role

As part of the Object-Oriented Programming course during the second year of university, I was tasked with developing a Java application called Gossip!. This project simulates the spread of gossip within a community, tracking who knows what about whom. The application allows users to create people, landmarks, and groups, and simulate the sharing of gossip under specific rules. I was responsible for the core logic, object-oriented design, and command-line interface.

Challenges & Strategy

The main challenge was implementing the complex rules of gossip spreading, including different types of people (forgetful, gossipers, and sealed lips) and their unique behaviors. Additionally, managing the dynamic nature of groups within landmarks and ensuring accurate gossip tracking required careful planning.

  • Solution: I used object-oriented programming principles to create classes for people, landmarks, groups, and gossips, ensuring modularity and reusability.
  • Solution: I implemented a command-line interface to handle user input and output, following the exact specifications provided in the project brief.
  • Solution: I conducted thorough testing to ensure that all edge cases, such as gossip sharing and group dynamics, were handled correctly.

Results & Impact

  • Functional Application: The final product was a fully functional Java program that could simulate the spread of gossip in a community, complete with all the rules and mechanics described in the project brief.
  • Positive Feedback: The project was well-received by the course instructors, who praised the clarity of the code and the robustness of the simulation logic.
  • Learning Experience: This project significantly improved my understanding of object-oriented design, Java programming, and problem-solving in a simulation context.
