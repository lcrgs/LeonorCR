ArtAuctions

Simulating an online auction system for art pieces.

Java Programming | Algorithms & Data Structures | Simulation

My Role

As part of the Algorithms and Data Structures course during the second year of university, I was tasked with developing a Java application called ArtAuctions. This project simulates an online auction system where users can register as collectors or artists, add artworks, and participate in auctions. I was responsible for the core logic, data structure design, and command-line interface.

Challenges & Strategy

The main challenge was implementing the complex rules of user roles, artwork management, and auction dynamics. Additionally, managing the dynamic nature of user interactions, artwork uploads, and auction bidding required careful planning.

  • Solution: I used object-oriented programming principles to create classes for users, artworks, auctions, and bids, ensuring modularity and reusability.
  • Solution: I implemented a command-line interface to handle user input and output, following the exact specifications provided in the project brief.
  • Solution: I conducted thorough testing to ensure that all edge cases, such as duplicate bids and auction closures, were handled correctly.

Results & Impact

  • Functional Application: The final product was a fully functional Java program that could simulate an online auction system, complete with all the rules and mechanics described in the project brief.
  • Positive Feedback: The project was well-received by the course instructors, who praised the clarity of the code and the robustness of the simulation logic.
  • Learning Experience: This project significantly improved my understanding of algorithms, data structures, and problem-solving in a simulation context.
